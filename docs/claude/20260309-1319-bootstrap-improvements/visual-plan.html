01 sync_paru Rewrite

File: bootstrap/configure — lines 51-128

Before — Current
Step 1 — Remove
Diff installed vs Parufile, auto-remove with --noconfirm
Step 2 — Orphans
Clean orphaned deps with --noconfirm
Step 3 — Update + Refresh
paru -Syu — refreshes repos AND upgrades all at once
Step 4 — Install
paru -S --needed --noconfirm — install new packages
Risks: Removal happens first (before you can even see what's new). Uses --noconfirm on destructive operations. No blank-line/comment support in Parufile.
After — Proposed
Step 1 — Refresh
paru -Sy — update package databases only, no upgrades
Step 2 — Install New
paru -S --needed — install packages from Parufile (skip existing)
Step 3 — Remove Stale
Diff installed vs Parufile+Parufile.system. Auto-remove with --noconfirm
Step 4 — Update
paru -Su — upgrade all outdated packages
Improvements: Repo refresh is separate from upgrades. Install happens before removal. Clear step ordering. Supports comments and blank lines.
Key Code Change — Step Reordering
  # Step 1: Refresh package databases (no upgrades yet)
  paru -Sy

  # Step 2: Install new packages from Parufile
  paru -S --needed "${desired_packages[@]}"

  # Step 3: Remove packages not in Parufile or Parufile.system
  if [ ${#packages_to_remove[@]} -gt 0 ]; then
    echo "Removing packages not in Parufile or Parufile.system..."
    paru -Rns --noconfirm "${packages_to_remove[@]}"
  fi

  # Step 4: Update all outdated packages
  paru -Su

02 Brew Sync Chain

File: bootstrap/configure — line 136

Before
bootstrap/configure:136
brew bundle check || brew bundle install && brew bundle cleanup --force && brew bundle dump --force
Operator precedence bug: && binds tighter than ||. If check succeeds, nothing else runs. If it fails, the entire right side executes as a chain. The check guard is misleading — install is already idempotent.
After
bootstrap/configure:136-138
brew bundle install
brew bundle cleanup --force
brew bundle dump --force
Three clear steps: Install missing → remove unlisted → regenerate Brewfile in sorted canonical format. No ambiguous operator precedence. Each step on its own line.

04 Harden install.sh

File: install.sh (top-level)

Before
#!/bin/sh

scripts_dir="$HOME/.scripts"
if [ -d "$scripts_dir" ]; then
  rm -rf "$scripts_dir"
fi

for dir in */; do
  pushd $dir > /dev/null
  if test -e "install.sh"; then
    ./install.sh
  fi
  popd > /dev/null
done
Issues: #!/bin/sh but uses bash-only pushd/popd. No set -e. Silent failures. No logging. Unquoted $dir.
After
#!/usr/bin/env bash
set -euo pipefail

scripts_dir="$HOME/.scripts"
if [ -d "$scripts_dir" ]; then
  rm -rf "$scripts_dir"
fi

failed=()
for dir in */; do
  if [ -e "$dir/install.sh" ]; then
    echo "Installing $dir..."
    if ! (cd "$dir" && ./install.sh); then
      echo "  FAILED: $dir"
      failed+=("$dir")
    fi
  fi
done

if [ ${#failed[@]} -gt 0 ]; then
  echo "Failed modules:"
  printf "  - %s\n" "${failed[@]}"
  exit 1
fi

08 Implementation Notes

Backward compatibility: All changes are backward-compatible. The sync and bootstrap function signatures are unchanged. Sourcing the configure script and calling functions works exactly as before.
Migration required: On macOS, run brew remove go gum hut pre-commit to remove Homebrew copies (or let brew bundle cleanup handle it on next sync). On Linux, uv will be auto-removed by sync_paru on next sync since it's no longer in the Parufile.
Design decision — mise @latest: Per-project .mise.toml files pin specific versions for each project. The global @latest provides a good system default for cross-project tools (gum, pre-commit, etc.). No change needed.
